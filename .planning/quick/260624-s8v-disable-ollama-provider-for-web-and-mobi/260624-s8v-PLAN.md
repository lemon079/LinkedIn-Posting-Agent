# Plan: Disable Ollama provider for web and mobile apps

We want to only allow selecting the local Ollama provider when the application is running inside a desktop app container (Tauri). On web browsers and Capacitor mobile apps, the Ollama option should be disabled.

## Proposed Changes

### UI & Settings Panel

#### [MODIFY] [SettingsPanel.tsx](file:///d:/Work/linkedin-agent/src/components/SettingsPanel.tsx)
- Add a helper check `isTauri` using `window.__TAURI__` or `window.__TAURI_INTERNALS__`.
- Under the AI Provider select options, conditionally render `<option value="ollama">Ollama</option>` only if `isTauri` is true.
- Add a `useEffect` hook to reset the provider to `"gemini"` if the loaded provider is `"ollama"` but `isTauri` is false.

#### [MODIFY] [useAgent.ts](file:///d:/Work/linkedin-agent/src/hooks/useAgent.ts)
- Add the `isTauri` check.
- In the settings load/initialization effect, if the loaded provider is `"ollama"` and `isTauri` is false, fallback to `"gemini"`.
