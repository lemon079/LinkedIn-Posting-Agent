# Plan: Disable settings access during post generation

We want to prevent the user from accessing the API/Account settings panel while a LinkedIn post is currently being generated.

## Proposed Changes

### UI Components

#### [MODIFY] [Header.tsx](file:///d:/Work/linkedin-agent/src/components/Header.tsx)
- Add a `disabled` prop of type `boolean` to `HeaderProps`.
- Bind the `disabled` state to the `<Button>` component so that the button becomes unclickable during generation.

#### [MODIFY] [page.tsx](file:///d:/Work/linkedin-agent/src/app/page.tsx)
- Pass `disabled={isGenerating}` from the main page state to the `Header` component.
- Guard the `onOpenSettings` function inside `page.tsx` so that settings cannot open if `isGenerating` is true.
