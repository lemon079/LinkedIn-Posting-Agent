# Codebase Structure

**Analysis Date:** 2026-06-17

## Directory Layout

```
linkedin-agent/
├── agent/                  # Specifications, rules and progress logs
│   ├── INSTRUCTIONS.md     # System rules and instructions
│   ├── PROGRESS.md         # Log of completed milestones and status
│   └── PROJECT.md          # Technical specifications and architecture
├── dist/                   # Compiled JavaScript output (generated)
├── src/                    # Application source files
│   ├── config/             # Environment validation and loader
│   │   └── env.ts          # Validates and exports system configuration
│   ├── core/               # Shared constants, prompts and schemas
│   │   ├── prompts.ts      # ghostwriter prompts and rules
│   │   └── state.ts        # LangGraph state annotations
│   ├── graph/              # LangGraph setup and individual nodes
│   │   ├── index.ts        # Graph compiler, edges, and checkpoints
│   │   └── nodes/          # Function handlers for each node
│   │       ├── generatePost.ts
│   │       ├── publishPost.ts
│   │       └── validatePost.ts
│   ├── services/           # External API callers
│   │   ├── linkedin.ts     # UGC post publish endpoint wrapper
│   │   └── llm.ts          # Instantiates ChatGoogle model
│   └── scheduler.ts        # CLI entry point, scheduler, and console interface
├── .env.example            # Environment variables template
├── eslint.config.js        # ESLint parser and rules configurations
├── package.json            # Dependencies, compiler scripts, and run commands
├── tsconfig.json           # TypeScript compiler configuration
└── README.md               # Quickstart guide
```

## Directory Purposes

**agent/**
- Purpose: Developer logs and technical requirements.
- Contains: Documentation describing features, instructions, and progress states.

**src/config/**
- Purpose: Application configuration.
- Contains: TypeScript configurations parsed from `.env`.

**src/core/**
- Purpose: Shared data annotations, types, and long prompts.
- Contains: Application constants and state definitions.

**src/graph/**
- Purpose: Orchestration definition and logic flow.
- Contains: Nodes, compiler definitions, conditional edges routing.
- Subdirectories: `nodes/` contains code for individual execution stages.

**src/services/**
- Purpose: Integration helpers communicating with third-party networks.
- Contains: Client wrappers for Gemini and LinkedIn REST API.

## Key File Locations

**Entry Points:**
- `src/scheduler.ts` - Interactive CLI runner and persistent cron process.

**Configuration:**
- `package.json` - Node module configuration.
- `tsconfig.json` - TypeScript module configuration (`module: NodeNext`).
- `eslint.config.js` - ESLint linter configuration.
- `.env` - Credentials (gitignored).

**Core Logic:**
- `src/graph/index.ts` - LangGraph compilation.
- `src/graph/nodes/generatePost.ts` - Draft generation logic.
- `src/services/linkedin.ts` - UGC post publishing logic.

## Naming Conventions

**Files:**
- camelCase for TypeScript modules (e.g. `generatePost.ts`, `publishPost.ts`, `validatePost.ts`, `scheduler.ts`).
- UPPERCASE for markdown documentation and configs (`README.md`, `PROGRESS.md`).

**Directories:**
- lowercase, singular or plural (e.g. `config`, `core`, `nodes`, `services`).

**Special Patterns:**
- Relative imports inside `src/` must specify the `.js` extension (e.g., `import { config } from "../config/env.js"`), due to NodeNext ESM specifications.

## Where to Add New Code

**New Agent Stage (Node):**
- Implementation: `src/graph/nodes/{nodeName}.ts`
- Registration: Import and add to builder in `src/graph/index.ts`. Add corresponding type declarations in `src/core/state.ts`.

**New API Wrapper:**
- Implementation: `src/services/{serviceName}.ts`

**New Environment Settings:**
- Definition: Define property in `AppConfig` interface inside `src/config/env.ts`, add validation checks, and update `.env.example`.

## Special Directories

**dist/**
- Purpose: Compiled JavaScript.
- Source: Generated by running `npm run build` (`tsc`).
- Committed: No (ignored in `.gitignore`).
